🐳 Podman Volumes Demo

📁 Volume Information

Web Content: This page is served from a volume

Logs: Nginx logs are stored in a named volume

Configuration: Nginx config is mounted from host

🏷️ Named Volumes

Managed by Podman, stored in: /var/lib/containers/storage/volumes/

Perfect for: Application data, databases, logs

🔗 Bind Mounts

Direct mapping from host filesystem to container

Perfect for: Development, configuration files, source code

💾 tmpfs Mounts

Temporary filesystem in memory

Perfect for: Temporary data, caches, sensitive information

📊 This demonstrates how Podman volumes work with persistent storage!

🧪 Test Commands

• Health check: curl http://localhost:8080/health

• View logs: podman logs <container-name>

• Inspect volumes: podman volume inspect <volume-name>
